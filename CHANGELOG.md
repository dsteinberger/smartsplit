# Changelog

All notable changes to SmartSplit will be documented in this file.

Format based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [Unreleased]

### Changed

- **Architecture pivot: Proxy Anticipateur** — SmartSplit is now a dual-mode proxy. Agent mode predicts and pre-executes read-only tool calls; API mode enriches with web search and pre-analysis.
- **Fast detector** replaces LLM-based triage — heuristic decision (<1ms) for TRANSPARENT vs ENRICH, no LLM call on most requests.
- **TRANSPARENT path** (55% of requests) — forward directly to brain, zero overhead.
- **ENRICH path** (45% of requests) — workers prepare context (web search, analysis, comparison), brain receives enriched prompt.
- Brain auto-detected from configured API keys (paid first: Claude > GPT > DeepSeek, then free: Cerebras > Groq > Gemini).
- Brain excluded from worker pool to prevent self-calls.
- Provider overrides per task type (`"overrides": {"code": "anthropic"}`).
- Dependency-aware subtask execution with context injection between dependent tasks.

### Added

- **HTTPS proxy mode** (`--proxy`) — transparent HTTPS interception for Claude Code and any agent. Uses your existing subscription auth, no API key needed. Normal quotas apply.
- **Tool anticipation** — 3-layer prediction engine (rules 0ms → learned patterns 0ms → LLM ~200ms) predicts read-only tool calls before the LLM asks. Pre-fetches files, grep results, web searches in parallel.
- **Tool-Aware Proxy** — compresses large tool results (grep, bash, web search, git) before forwarding to the brain. Three categories: simple (passthrough), smart (compress if > 3000 chars), decisional (never touch).
- **Tool Pattern Learner** — learns from actual tool calls using Wilson score confidence across 5 pattern types. Staleness decay (7 days halve, 30 days evict). JSON persistence at `~/.smartsplit/tool_patterns.json`.
- **`setup-claude` command** — one-time setup helper that generates CA certs and prints launch instructions.
- **Multilingual keyword detection** — domain detection and enrichment triage now work in 9 languages: EN, FR, ES, PT, DE, ZH, JA, KO, RU. Translations managed via `smartsplit/i18n_keywords.py`, generated by `scripts/generate_i18n.py`.
- **Language-aware responses** — enriched prompts instruct the brain to reply in the user's language.
- `call_brain()` in ProviderRegistry with automatic fallback.
- `detect()` heuristic detector — checks tool calls, prompt length, keywords, conversation history.
- 5 enrichment types: web_search, pre_analysis, multi_perspective, context_summary, verification.
- Brain/worker status in `/health` and `/metrics` endpoints.
- `.editorconfig`, `.github/CODEOWNERS`, expanded `.pre-commit-config.yaml`.
- `pipx install smartsplit` as recommended install method.

### Fixed

- **Deep copy in enrichment** — `_build_enriched_messages()` now uses `copy.deepcopy()` to prevent corrupting original messages used by fallback.
- **Truncation bug** — `_truncate_messages()` no longer keeps 500 chars when budget is exhausted.
- **Single-provider setups** — `call_free_llm()` now falls back to brain when no free LLM is available.
- **Circuit breaker** — raised threshold to 5 failures, added exponential backoff (30s→60s→…→30min), success resets backoff.
- **Thread safety in MAB** — `BanditScorer.record()` and `flush()` protected by lock; data validated on load.
- **Quality gate** — reduced error scan to 80 chars (was 200) so "I apologize… Here's the code" is not rejected; raised repetition threshold.
- **`extract_prompt()`** — returns empty string when no user message found (was returning last message regardless of role).
- **Confidence scoring** — capped at 1.0 to prevent unbounded scores.
- **Provider starvation** — strong models now get `cost_score=0.2` (was 0.0), fast models `0.5` (was 0.3).
- **Provider call timeout** — 30s per-call timeout prevents hanging providers from blocking routing.
- **Fast model MAB** — fast-tier fallback now records success in the bandit scorer.
- **Context injection logging** — failure and skip reasons now logged explicitly.
- Race condition in QuotaTracker concurrent access.
- `depends_on` index validation after subtask truncation.
- Deep copy messages between subtasks to prevent shared mutable state.
- JSON parsing error handling in all providers.
- CircuitBreaker dict mutation safety (`del` → `pop`).
- Narrower exception catching in planner (no more broad `except Exception`).
- Deduplicated `_EMPTY_USAGE` and `OpenAIUsage` / `TokenUsage` models.

## [0.1.0] - 2026-04-07

### Added

- OpenAI-compatible HTTP endpoint (`/v1/chat/completions`)
- LLM-based triage: RESPOND (route to best LLM) and ENRICH (web search + route). Language-agnostic with keyword fallback
- Prompt decomposition into subtasks with domain detection
- Intelligent routing: scores each provider per task type (code, reasoning, translation, etc.)
- 8 free LLM providers: Groq, Cerebras, Gemini, OpenRouter, Mistral, HuggingFace, Cloudflare (+ DeepSeek paid)
- 2 optional paid LLM providers: Anthropic, OpenAI (disabled by default)
- 2 search providers: Serper, Tavily
- Real token usage tracking (aggregated across providers, returned in OpenAI format)
- Circuit breaker per provider (3 failures → 30 min cooldown)
- Quality gates with refusal detection and auto-escalation
- Decompose cache (LRU, 24h TTL)
- Fast/strong model tiers for paid providers (Haiku/Sonnet, GPT-4o-mini/GPT-4o)
- Automatic tier selection based on task complexity
- Full conversation context preservation (messages passed to LLMs, not just last prompt)
- Context summary injection for multi-subtask decomposition
- HTTP 503 with structured error when all providers are unavailable
- Metrics endpoint (`/metrics`) with triage stats, savings, cache hit rate
- Health endpoint (`/health`) and savings report (`/savings`)
- Docker support with lightweight image, HEALTHCHECK, and Docker Compose
- Docker image published to GitHub Container Registry on release
- CLI with mode selection (`--mode economy|balanced|quality`)
- Configuration via environment variables or JSON config file
- Daily Provider Watch job (automated LLM ecosystem monitoring via GitHub Actions)
- GitHub issue templates, PR template, Dependabot, pre-commit hooks
- Adaptive scoring via MAB (UCB1) — learns which providers work best from real usage
- Codecov integration for test coverage tracking
- Makefile with common commands (`make check`, `make run`, `make docker-build`, etc.)
- Example config files for OpenCode, Aider, Continue in `examples/`
